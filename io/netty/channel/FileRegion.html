io.netty.channel

Interface FileRegion

  • All Superinterfaces:
    ReferenceCounted
    所有已知实现类：
    DefaultFileRegion 

    public interface FileRegion
extends ReferenceCounted
    通过支持zero-copy file transfer的Channel发送的文件区域。

    升级您的JDK / JRE

    FileChannel.transferTo(long, long, WritableByteChannel)在旧版本的Sun JDK及其衍生版本中至少有四个已知的错误。 如果您打算使用零拷贝文件传输，请将您的JDK升级到1.6.0_18或更高版本。
    • 5103988 - FileChannel.transferTo（）应该为EAGAIN返回-1，而不是抛出IOException
    • 6253145 - 超出2GB边界时，Linux上的FileChannel.transferTo（）失败
    • 6427312 - FileChannel.transferTo（）抛出IOException“系统调用中断”
    • 6470086 - FileChannel.transferTo（2147483647，1，channel）导致“值太大”异常

    检查您的操作系统和JDK / JRE

    如果您的操作系统（或JDK / JRE）不支持零拷贝文件传输，则发送包含FileRegion的文件可能会失败或性能降低。 例如，发送大文件在Windows中运行不正常。

    并非所有的传输都支持它
