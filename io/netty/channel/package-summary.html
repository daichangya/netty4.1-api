Package io.netty.channel

核心频道API是异步和事件驱动的各种传输（如 NIO Channel）的抽象 。

See: 描述

    AddressedEnvelope<M,A extends java.net.SocketAddress>
    包含发件人地址和收件人地址的其他邮件的消息。
    Channel
    与网络套接字或能够进行I / O操作（如读取，写入，连接和绑定）的组件的关系。

    不应该从用户代码调用的 不安全操作。

    听取ChannelFuture的结果。
    ChannelHandler
    处理I / O事件或拦截I / O操作，并将其转发到其ChannelPipeline中的下一个处理程序。
    ChannelHandlerContext
    使ChannelHandler能够与其ChannelPipeline和其他处理程序进行交互。
    ChannelId
    表示Channel的全局唯一标识符。
    ChannelInboundHandler
    ChannelHandler ，它为状态更改添加回调。

    ChannelHandler将通知IO出站操作。
    ChannelOutboundInvoker  
    ChannelPipeline
    处理或拦截Channel的入站事件和出站操作的ChannelHandler列表 。
    ChannelProgressiveFuture
    用于指示FileRegion传输进度的特殊ChannelFuture
    ChannelProgressiveFutureListener
    一个 EventListener监听器，一旦与未来相关的发送任务正在被传送，该监听器将被调用。
    ChannelProgressivePromise
    特殊的ChannelPromise ，一旦关联的字节被传输，它将被通知。
    ChannelPromise
    特别ChannelFuture这是可写的。
    EventLoop
    一旦注册，将处理Channel的所有I / O操作。
    EventLoopGroup
    特殊EventExecutorGroup ，允许注册Channel ，在事件循环期间进行处理以供以后选择。
    FileRegion
    通过支持zero-copy file transfer的Channel发送的文件区域。
    MaxBytesRecvByteBufAllocator
    RecvByteBufAllocator ，其基于每次读取的最大值和当事件循环尝试读取操作时的最大量来限制读取操作。
    MaxMessagesRecvByteBufAllocator
    RecvByteBufAllocator ，它限制事件循环尝试执行读取操作时将要尝试的读取操作数。
    MessageSizeEstimator
    负责估算消息的大小。
    MessageSizeEstimator.Handle  
    RecvByteBufAllocator
    分配一个新的接收缓冲区，其容量可能足够大，可以读取所有入站数据，并且足够小，不会浪费其空间。
    RecvByteBufAllocator.ExtendedHandle  
    RecvByteBufAllocator.Handle Deprecated
    使用RecvByteBufAllocator.ExtendedHandle 。
    SelectStrategy
    选择策略界面。
    SelectStrategyFactory
    每次创建新的SelectStrategy的工厂。
    ServerChannel
    A Channel接受传入的连接尝试并通过接受它们来创建其子Channel 。

    表示Channel实现的属性。
    ChannelOption<T>
    A ChannelOption允许以类型安全的方式配置ChannelConfig 。

    ChannelPromiseAggregator Deprecated
    使用PromiseCombiner类别，通过收听单个期货并在所有期货完成时产生汇总结果（成功/失败），用于将多个渠道期货合并为一个。
    ChannelPromiseNotifier
    ChannelFutureListener实现需要其他ChannelPromise （s）并在完成时通知它们。
    CoalescingBufferQueue
    生产者通过反复添加ByteBuf来添加字节的FIFO队列，消费者以任意长度获取字节。
    CombinedChannelDuplexHandler<I extends ChannelInboundHandler,O extends ChannelOutboundHandler>
    将ChannelInboundHandler和ChannelOutboundHandler组合成一个ChannelHandler 。

    DefaultChannelConfig
    默认的ChannelConfig实现。
    DefaultChannelId
    默认的ChannelId实现。
    DefaultChannelPipeline
    默认的ChannelPipeline实现。
    DefaultChannelProgressivePromise
    默认的ChannelProgressivePromise实现。
    DefaultChannelPromise
    默认的ChannelPromise实现。
    DefaultEventLoop  
    DefaultEventLoopGroup
    MultithreadEventLoopGroup必须用于当地运输。
    DefaultFileRegion
    从FileChannel或File传输数据的默认FileRegion实现。
    DefaultMaxBytesRecvByteBufAllocator
    RecvByteBufAllocator基于从每次读取的最大字节数中减去该值来产生缓冲区大小预测。
    DefaultMaxMessagesRecvByteBufAllocator
    MaxMessagesRecvByteBufAllocator的默认实现，它尊重ChannelConfig.isAutoRead()并防止溢出。
    DefaultMessageSizeEstimator
    默认MessageSizeEstimator实施支持的尺寸的估计ByteBuf ， ByteBufHolder和FileRegion 。
    DefaultSelectStrategyFactory
    使用默认选择策略的工厂。
    DelegatingChannelPromiseNotifier  
    FixedRecvByteBufAllocator
    总是产生相同缓冲区大小预测的RecvByteBufAllocator 。
    MultithreadEventLoopGroup
    EventLoopGroup实现的抽象基类， 它们同时处理具有多个线程的任务。
    PendingWriteQueue
    等待稍后执行的写入操作队列。
    PreferHeapByteBufAllocator
    包裹另一个ByteBufAllocator并在除了明确请求直接缓冲区外的地方使用heapbuffers。
    RecvByteBufAllocator.DelegatingHandle
    一个RecvByteBufAllocator.Handle代表所有呼叫到其他RecvByteBufAllocator.Handle 。
    ReflectiveChannelFactory<T extends Channel>
    一个ChannelFactory实例化一个新Channel通过反射性地调用它的默认构造函数。
    SimpleChannelInboundHandler<I>

    SingleThreadEventLoop
    EventLoop的抽象基类，它在单个线程中执行其所有提交的任务。
    ThreadPerChannelEventLoop
    SingleThreadEventLoop用于处理OIO Channel的。
    ThreadPerChannelEventLoopGroup
    一个EventLoopGroup创建一个EventLoop每Channel 。
    VoidChannelPromise  
    WriteBufferWaterMark
    WriteBufferWaterMark用于为写入缓冲区设置低水位标志和高位标志。
  • 异常摘要  
    异常 描述
    ChannelException
    当I / O操作失败时引发的 RuntimeException 。
    ChannelPipelineException
    A ChannelException当ChannelPipeline未能执行操作时引发。
    ConnectTimeoutException
    ConnectException如果由于连接超时而无法建立连接，将会抛出它。
    EventLoopException
    Special ChannelException当发生错误时将由EventLoop和EventLoopGroup实现抛出。

    Annotation Type 描述
    ChannelHandler.Sharable
    表示可以将带有注释的ChannelHandler的同一个实例多次添加到一个或多个ChannelPipeline ，而不会出现竞争条件。
