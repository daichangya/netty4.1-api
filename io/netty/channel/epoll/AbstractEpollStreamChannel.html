      • spliceTo

        public final ChannelFuture spliceTo(FileDescriptor ch,
                                    int offset,
                                    int len)
        从AbstractEpollStreamChannel拼接到另一个FileDescriptor 。 该offset针对偏移FileDescriptor和len是字节拼接的数量。 如果使用Integer.MAX_VALUE它将拼接，直到ChannelFuture被取消或失败。 请注意：
        • EpollChannelConfig.getEpollMode()必须是EpollMode.LEVEL_TRIGGERED这个AbstractEpollStreamChannel
        • ChannelFuture通知后， FileDescriptor将不会关闭
        • 此通道必须注册到事件循环或IllegalStateException将被抛出。

      • spliceTo

                                    int offset,

        • EpollChannelConfig.getEpollMode()必须EpollMode.LEVEL_TRIGGERED这个AbstractEpollStreamChannel

      • doWriteSingle

        protected int doWriteSingle(ChannelOutboundBuffer in)
                     throws java.lang.Exception
        尝试编写单个对象。
        参数
        in - 包含要写入对象的集合。
        结果
        应该从ChannelConfig.getWriteSpinCount()开始的写入量值递减的值。 典型的用例如下：
        • 0 - 如果没有尝试写入。 如果遇到空的ByteBuf （或其他空的内容），这是适当的
        • 1 - 如果向OS写入数据的单个调用
        • ChannelUtils.WRITE_STATUS_SNDBUF_FULL - 如果尝试向操作系统写入数据，但未接受任何数据
        异常
        java.lang.Exception - 如果发生I / O错误。
