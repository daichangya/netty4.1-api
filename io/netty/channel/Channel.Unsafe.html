io.netty.channel

Interface Channel.Unsafe

  • All Known Subinterfaces:
    AbstractNioChannel.NioUnsafe
    所有已知实现类：
    AbstractChannel.AbstractUnsafe ， AbstractNioByteChannel.NioByteUnsafe ， AbstractNioChannel.AbstractNioUnsafe
    Enclosing interface:
    Channel 

    public static interface Channel.Unsafe
    不应该从用户代码调用的不安全操作。 这些方法仅用于实现实际的传输，并且必须从I / O线程调用，但以下方法除外：
    • localAddress()
    • remoteAddress()
    • closeForcibly()
    • register(EventLoop, ChannelPromise)
    • deregister(ChannelPromise)
    • voidPromise()

    • 方法详细信息

      • recvBufAllocHandle

        RecvByteBufAllocator.Handle recvBufAllocHandle()
        返回分配的RecvByteBufAllocator.Handle ，用于在接收数据时分配ByteBuf 。

      • localAddress

        java.net.SocketAddress localAddress()
        返回本地绑定的 null如果没有则返回 SocketAddress 。

      • remoteAddress

        java.net.SocketAddress remoteAddress()
        返回 SocketAddress其上结合远程或 null如果未尚未绑定。

      • register

        void register(EventLoop eventLoop,
              ChannelPromise promise)
        注册ChannelPromise的Channel，并在注册完成后通知ChannelFuture 。

      • bind

        void bind(java.net.SocketAddress localAddress,
          ChannelPromise promise)
        绑定SocketAddress到Channel的的ChannelPromise并通知其一旦其完成。

      • connect

        void connect(java.net.SocketAddress remoteAddress,
             java.net.SocketAddress localAddress,
             ChannelPromise promise)
        连接Channel指定的ChannelFuture与指定的远程SocketAddress 。 如果应该使用特定的本地SocketAddress则需要将其作为参数给出。 否则，只需将null传递给它。 一旦连接操作完成， ChannelPromise将会收到通知。

      • disconnect

        void disconnect(ChannelPromise promise)
        断开ChannelFuture的Channel，并在操作完成后通知ChannelPromise 。

      • close

        void close(ChannelPromise promise)
        关闭ChannelPromise的Channel，并在操作完成后通知ChannelPromise 。

      • closeForcibly

        void closeForcibly()
        立即关闭Channel ，但不会触发任何事件。 可能仅在注册尝试失败时才有用。

      • deregister

        void deregister(ChannelPromise promise)
        操作完成后，从EventLoop注销ChannelPromise的Channel ，并通知ChannelPromise 。

      • beginRead

        void beginRead()
        附表填充第一的入缓冲区的读取操作ChannelInboundHandler在ChannelPipeline 。 如果已经有待处理的读取操作，则此方法不执行任何操作。

      • write

        void write(java.lang.Object msg,
           ChannelPromise promise)
        计划写入操作。

      • flush

        void flush()
        清除通过 write(Object, ChannelPromise)计划的所有写入操作。

      • voidPromise

        ChannelPromise voidPromise()
        返回一个特殊的ChannelPromise，可以重复使用并传递给Channel.Unsafe中的操作。 它永远不会被通知成功或错误，所以只是一个以ChannelPromise作为参数但不想获得通知的操作的占位符。

      • outboundBuffer

        ChannelOutboundBuffer outboundBuffer()
        返回Channel中存储未决写入请求的ChannelOutboundBuffer 。
