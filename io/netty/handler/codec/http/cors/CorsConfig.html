      • isNullOriginAllowed

        public boolean isNullOriginAllowed()
        如果从本地文件系统加载资源，Web浏览器可能会将'Origin'请求标头设置为'null'。 如果isNullOriginAllowed为true，则服务器将响应CORS响应头“Access-Control-Allow-Origin”的通配符。
        结果
        true如果应该支持'null'来源。

      • exposedHeaders

        public java.util.Set<java.lang.String> exposedHeaders()
        返回一组要显示给呼叫客户端的头文件。 在简单的CORS请求期间，浏览器只能使用某些响应头，例如使用： 
          xhr.getResponseHeader("Content-Type");
        默认情况下可用的标题是：
        • 缓存控制
        • 内容语言
        • 内容类型
        • 过期
        • 上一次更改
        • 附注
        为了公开其他头文件，需要指定它们，这是通过将头文件名称添加到CORS的Access-Control-Expose-Headers响应头文件所实现的。
        结果
        List<String>要公开的标题列表。

      • isCredentialsAllowed

        public boolean isCredentialsAllowed()
        确定CORS请求是否支持Cookie。 默认情况下，Cookie不包括在CORS请求中，但如果isCredentialsAllowed返回true，Cookie将被添加到CORS请求中。 将此值设置为true会将CORS的“访问控制 - 允许 - 凭据”响应标头设置为true。 请注意，客户端也需要启用cookie支持。 客户需要选择通过以下方式发送cookie： 
          xhr.withCredentials = true;
        'withCredentials'的默认值为false，在这种情况下，不发送cookie。 如果将此设置为true，则会在交叉源请求中包含Cookie。
        结果
        true如果支持cookies。

      • maxAge

        public long maxAge()
        获取maxAge设置。 当进行预检请求时，客户端必须执行两个请求，效率可能很低。 此设置将设置CORS的“Access-Control-Max-Age”响应标题，并启用预检响应在指定时间的高速缓存。 在此期间，不会进行预检请求。
        结果
        long可以高速缓存预检请求的时间（以秒为单位）。

      • allowedRequestMethods

        结果
        代表允许的请求方法的Set个HttpMethod个 。

      • allowedRequestHeaders

        结果

      • isShortCircuit

        public boolean isShortCircuit()
        确定CORS请求在进一步处理之前是否应该被拒绝。 在处理请求后设置CORS头。 这可能并不总是需要的，这个设置将检查Origin是否有效，如果它不是有效的，则不会进行进一步的处理，并且错误将被返回给呼叫客户端。
        结果
        true如果一个CORS请求在接收到一个无效的Origin头部时应该短路。
