Interface InterfaceHttpPostRequestDecoder

  • 所有已知实现类：
    HttpPostMultipartRequestDecoder ， HttpPostRequestDecoder ， HttpPostStandardRequestDecoder 

    public interface InterfaceHttpPostRequestDecoder
    该解码器将解码Body并可以处理POST BODY。 完成后您必须致电destroy()以释放所有资源。

      • isMultipart

        boolean isMultipart()
        如果此请求是多部分请求，则为真
        结果
        如果此请求是多部分请求，则为真

      • setDiscardThreshold

        void setDiscardThreshold(int discardThreshold)
        设置缓冲区中的读取字节应该丢弃的字节数量。 设置这个较低值会降低内存使用量，但会增加更多内存副本的开销。 使用0来禁用它。

      • getDiscardThreshold

        int getDiscardThreshold()
        以字节为单位返回阈值，之后应丢弃缓冲区中的读取数据。

      • getBodyHttpDatas

        java.util.List<InterfaceHttpData> getBodyHttpDatas()
        这个getMethod从body返回一个所有HttpDatas的列表。
        如果分块，则必须使用offer（）getMethod提供所有块。 如果不是，则会引发NotEnoughDataDecoderException。
        结果
        POST getMethod的正文部分中的HttpDatas列表
        异常
        HttpPostRequestDecoder.NotEnoughDataDecoderException - 需要更多的块

      • getBodyHttpDatas

        java.util.List<InterfaceHttpData> getBodyHttpDatas(java.lang.String name)
        这个getMethod返回一个所有HttpDatas的列表，其中包含正文中给定的名字。
        如果分块，则必须使用offer（）getMethod提供所有块。 如果不是，则会引发NotEnoughDataDecoderException。
        结果
        具有给定名称的所有Body HttpDatas（忽略大小写）
        异常
        HttpPostRequestDecoder.NotEnoughDataDecoderException - 需要更多的块

      • getBodyHttpData

        InterfaceHttpData getBodyHttpData(java.lang.String name)
        这个getMethod从body返回具有给定名称的第一个InterfaceHttpData。
        如果分块，则必须使用offer（）getMethod提供所有块。 如果不是，则会引发NotEnoughDataDecoderException。
        结果
        具有给定名称的第一个Body InterfaceHttpData（忽略大小写）
        异常
        HttpPostRequestDecoder.NotEnoughDataDecoderException - 需要更多的块

      • offer

        InterfaceHttpPostRequestDecoder offer(HttpContent content)
        从新块初始化内部
        参数
        content - 新收到的块
        异常
        HttpPostRequestDecoder.ErrorDataDecoderException - 如果字符集解码或其他错误有问题

      • hasNext

        boolean hasNext()
        如果在当前的getStatus中，则为真，则存在来自Body的可用解码的InterfaceHttpData。 这个getMethod适用于分块和不分块的请求。
        结果
        如果在当前的getStatus中，则为true，则有解码的InterfaceHttpData
        异常
        HttpPostRequestDecoder.EndOfDataDecoderException - 没有更多数据可用

      • next

        InterfaceHttpData next()
        返回下一个可用的InterfaceHttpData;如果在调用时没有可用的InterfaceHttpData，则返回null。 随后的提供呼叫（httpChunk）可以启用更多数据。 完成处理后务必致电ReferenceCounted.release() ，以确保不泄漏任何资源
        结果
        下一个可用的InterfaceHttpData;如果没有，则返回null
        异常
        HttpPostRequestDecoder.EndOfDataDecoderException - 没有更多数据可用

      • currentPartialHttpData

        InterfaceHttpData currentPartialHttpData()
        如果当前处于解码状态（表示所有数据尚未处于内），则返回当前的InterfaceHttpData;如果当前没有处于解码状态的InterfaceHttpData（因为尚未解码或当前未解码），则返回null。 通过hasNext（）和next（）方法可以完全解码。
        结果
        如果当前处于解码状态，则返回当前的InterfaceHttpData;如果没有，则返回null。
