    • 方法详细信息

      • incrementAndGetNextStreamId

        int incrementAndGetNextStreamId()
        递增并获取下一个生成的流ID这个端点。 如果为负数，则此端点的流ID将耗尽，并且不会创建更多流。

      • isValidStreamId

        boolean isValidStreamId(int streamId)
        指示给定的streamId是否来自此端点用来创建新流的一组ID。

      • mayHaveCreatedStream

        boolean mayHaveCreatedStream(int streamId)
        指示此端点是否可能创建了给定的流。 这是true如果isValidStreamId(int)和streamId <= lastStreamCreated() 。

      • created

        boolean created(Http2Stream stream)
        指示此端点是否创建了给定的流。

      • canOpenStream

        boolean canOpenStream()
        指示是否可以在不违反 maxActiveStreams()的情况下打开由此端点创建的流。

      • createStream

        Http2Stream createStream(int streamId,
                         boolean halfClosed)
                  throws Http2Exception
        创建由此端点启动的流。 这可能会失败，原因如下：
        • 请求的流ID不是此端点的下一个顺序ID。
        • 流已经存在。
        • canOpenStream()是false 。
        • 连接被标记为消失。

        在通知Http2Connection.Listener之前，流的初始状态将立即设置。 状态转换对halfClosed敏感，由Http2Stream.open(boolean)定义。

        参数
        streamId - 流的ID
        halfClosed - 见 Http2Stream.open(boolean) 。
        异常
        Http2Exception
        另请参见：
        Http2Stream.open(boolean)

      • reservePushStream

        Http2Stream reservePushStream(int streamId,
                              Http2Stream parent)
                       throws Http2Exception
        在此端点的保留状态中创建推送流并通知所有监听者。 这可能会失败，原因如下：
        • 服务器推送不允许到相反的端点。
        • 所请求的流ID不是此端点的下一个顺序流ID。
        • 并发流的数量高于此端点允许的阈值。
        • 连接被标记为消失。
        • 父流ID不存在或不是从发送推送承诺的那一方的OPEN 。
        • 无法为新的流设置有效的优先级。
        参数
        streamId - 推送流的ID
        parent - 用于启动推送流的父流。
        异常
        Http2Exception

      • isServer

        boolean isServer()
        指示此端点是否是连接的服务器端。

      • allowPushTo

        void allowPushTo(boolean allow)
        这是从相反端点发送的值SETTINGS_ENABLE_PUSH 。 这种方法只能由Netty（不是用户）作为接收SETTINGS帧的结果来SETTINGS 。

      • allowPushTo

        boolean allowPushTo()
        这是从相反端点发送的值SETTINGS_ENABLE_PUSH 。 客户端端点的初始值必须为true ，并且服务器端点的值始终为false。

      • numActiveStreams

        int numActiveStreams()
        获取由此端点创建的活动流的数量（即 OPEN或 HALF CLOSED ）。

      • maxActiveStreams

        int maxActiveStreams()
        获取允许同时处于活动状态的最大数量的流（由此端点创建）。 这是从相反端点发送的SETTINGS_MAX_CONCURRENT_STREAMS值，用于限制由此端点创建的流。

        此方法返回的默认值必须是“无限”。

      • maxActiveStreams

        void maxActiveStreams(int maxActiveStreams)
        设置 SETTINGS_MAX_CONCURRENT_STREAMS的限制。
        参数
        maxActiveStreams - 允许同时激活的最大数量的流（由此端点创建）。 这是从对端发送的SETTINGS_MAX_CONCURRENT_STREAMS值，用于限制由此端点创建的流。

      • lastStreamCreated

      • lastStreamKnownByPeer

        int lastStreamKnownByPeer()
        如果此端点收到GOAWAY，这将是GOAWAY帧中的最后一个流ID。 否则，这将是-1 。
