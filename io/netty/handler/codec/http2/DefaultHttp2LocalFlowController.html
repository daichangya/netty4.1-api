    @UnstableApi

extends java.lang.Object
implements Http2LocalFlowController
    基本实现Http2LocalFlowController 。

    这个类不是线程安全的。 假设是所有方法都必须从单个线程中调用。 通常这个线程是由这个类管理的ChannelHandlerContext的事件循环线程。

      • incrementWindowSize

        public void incrementWindowSize(Http2Stream stream,
                                int delta)
                         throws Http2Exception
        从界面复制的描述： Http2FlowController
        按给定的增量增加流的流量控制窗口的大小。

        在Http2RemoteFlowController的情况下，在从远程端点接收到WINDOW_UPDATE帧时调用这个帧，以反映窗口大小的变化。

        对于Http2LocalFlowController，可以调用此方法来请求扩展由此端点发布的窗口大小。 然而，到WINDOW_UPDATE实际发送的时间由实施决定。

        Specified by:
        incrementWindowSize在界面 Http2FlowController
        参数
        stream - 主题流。 使用Http2Connection.connectionStream()来请求连接窗口的大小。
        delta - 流量控制窗口大小的变化。
        异常
        Http2Exception - 如果发生与协议有关的错误，则抛出。

      • consumeBytes

        public boolean consumeBytes(Http2Stream stream,
                            int numBytes)
                     throws Http2Exception
        从界面复制而来的描述： Http2LocalFlowController
        表示应用程序已经为给定流消耗了大量字节，因此准备从远程端点接收更多数据。 应用程序必须使用它接收到的任何字节，否则流控制窗口将崩溃。 消耗字节使流量控制器能够发送WINDOW_UPDATE以恢复流的一部分流量控制窗口。

        如果stream是null或关闭（即Http2Stream.state()方法返回Http2Stream.State.CLOSED ），则调用此方法不起作用。
        Specified by:
        consumeBytes在界面 Http2LocalFlowController
        参数
        stream - 应该释放窗口空间的流。 不能使用连接流对象。 如果stream是null或关闭（即Http2Stream.state()方法返回Http2Stream.State.CLOSED ），则调用此方法不起作用。
        numBytes - 要返回到流量控制窗口的字节数。
        结果
        如果发送 WINDOW_UPDATE则返回true，否则返回false。
        异常
        Http2Exception - 如果返回的字节数超过流的 Http2LocalFlowController.unconsumedBytes(Http2Stream) 。

        窗口更新比率用于确定何时必须发送窗口更新。 如果自上次更新以来处理的字节比例达到或超过此比例，则会发送窗口更新。 此窗口更新比率仅适用于streamId 。

        参数

        异常
        Http2Exception - 如果在生成 WINDOW_UPDATE帧时发生协议错误

      • receiveFlowControlledFrame

        public void receiveFlowControlledFrame(Http2Stream stream,
                                       ByteBuf data,
                                       int padding,
                                       boolean endOfStream)
                                throws Http2Exception
        从界面复制的描述： Http2LocalFlowController
        从远程端点DATA站DATA帧，并为stream以及连接应用流量控制策略。 如果任何流量控制策略被违反，则立即引发异常，否则该帧被认为具有“通过”流量控制。

        如果stream为null或关闭，则流量控制只应用于连接窗口，并立即消耗字节。
        Specified by:
        receiveFlowControlledFrame在界面 Http2LocalFlowController
        参数
        stream - 接收帧的主题流。 不能使用连接流对象。 如果stream为null或关闭，则只应将流量控制应用于连接窗口，并立即消耗字节。
        data - 帧的有效载荷缓冲区。
        padding - 应添加的其他字节以掩盖真实的内容大小。 必须介于0和256（含）之间。
        endOfStream - 指示这是否是来自此流的远程端点的最后一个帧。
        异常
        Http2Exception - 遇到任何流量控制错误。
