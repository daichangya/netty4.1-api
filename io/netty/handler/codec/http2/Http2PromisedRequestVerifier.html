Interface Http2PromisedRequestVerifier


  • @UnstableApi
public interface Http2PromisedRequestVerifier
    为用户定义推送请求的有效性提供可扩展性点。
    另请参见：
    [RFC http2], Section 8.2 。

    • 方法详细信息

      • isAuthoritative

        boolean isAuthoritative(ChannelHandlerContext ctx,
                        Http2Headers headers)
        确定Http2Headers是否对特定ChannelHandlerContext具有权威性 。
        参数
        ctx - 收到的 headers的上下文。
        headers - 要验证的标题。
        结果
        true如果 ctx是权威为 headers ， false其他。
        另请参见：
        [RFC http2], Section 10.1 。

      • isCacheable

        boolean isCacheable(Http2Headers headers)
        确定请求是否可缓存。
        参数
        headers - 推送请求的标头。
        结果
        true如果与 headers相关的请求已知可缓存，则 false否则 false 。
        另请参见：
        [RFC 7231], Section 4.2.3 。

      • isSafe

        boolean isSafe(Http2Headers headers)
        确定请求是否安全。
        参数
        headers - 推送请求的标题。
        结果
        true如果与 headers相关的请求已知是安全的， false否则。
        另请参见：
        [RFC 7231], Section 4.2.1 。
