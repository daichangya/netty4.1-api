io.netty.handler.codec.http2

Interface Http2RemoteFlowController.FlowControlled

  • 所有已知实现类：
    DefaultHttp2ConnectionEncoder.FlowControlledBase
    Enclosing interface:
    Http2RemoteFlowController 

    public static interface Http2RemoteFlowController.FlowControlled
    这个接口的实现用来渐进地将底层载荷的块写入流中。 如果write(io.netty.channel.ChannelHandlerContext, int)已被调用至少一次，且size()现在为零，则认为有效负载被完全写入。

    • 方法详细信息

      • size

        int size()
        按照应用于流量控制窗口的字节计算有效负载的大小。 一些有效载荷如HEADER帧对于流量控制没有成本，并且对于该值将返回0，即使它们HEADER产生非零数量的字节。 其他帧如DATA帧的有效载荷和填充数均与流量控制相比较。

      • error

        void error(ChannelHandlerContext ctx,
           java.lang.Throwable cause)
        被调用以指示在可以完全写入此对象之前发生错误。

        Http2RemoteFlowController将对此方法或writeComplete()进行一次调用。

        参数
        ctx - 由于错误而需要发生任何通信时使用的上下文。 如果连接尚未建立时发生异常，则这可能是null 。
        cause - 的错误。

      • writeComplete

        void writeComplete()
        在此对象成功写入后调用。

        Http2RemoteFlowController只会对此方法或error(ChannelHandlerContext, Throwable)进行一次调用。

      • write

        void write(ChannelHandlerContext ctx,
           int allowedBytes)
        将最多allowedBytes封装的有效载荷写入流中。 请注意，可以传递一个值0，这将允许写入流量控制大小为== 0的有效载荷。 流量控制器可以使用不同的值多次调用此方法，直到有效负载完全写入，即写入为0后的大小。

        当发生异常时， Http2RemoteFlowController将拨打error(ChannelHandlerContext, Throwable) 。

        参数
        ctx - 用于书写的上下文。
        allowedBytes - 此时有效负载可写入的字节数的上限。

      • merge

        boolean merge(ChannelHandlerContext ctx,
              Http2RemoteFlowController.FlowControlled next)
        将next消息的内容合并到此消息中，以便它们可以作为一个单元写出。 这允许将许多小消息写入单个数据帧。
        结果
        true如果 next已成功合并且不需要入队， false否则。
