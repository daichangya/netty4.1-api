io.netty.handler.codec.http2

Interface Http2FrameWriter

  • All Superinterfaces:
    java.lang.AutoCloseable，java.io.Closeable， Http2DataWriter
    All Known Subinterfaces:
    Http2ConnectionEncoder
    所有已知实现类：
    CompressorHttp2ConnectionEncoder ， DecoratingHttp2ConnectionEncoder ， DecoratingHttp2FrameWriter ， DefaultHttp2ConnectionEncoder ， DefaultHttp2FrameWriter ， Http2OutboundFrameLogger ， StreamBufferingEncoder 

    @UnstableApi
public interface Http2FrameWriter
extends Http2DataWriter, java.io.Closeable
    负责将HTTP / 2帧封送到频道的作者。 该接口中的所有写入方法都写入上下文，但不要刷新。 要执行刷新，您必须单独拨打ChannelHandlerContext.flush() 。

      • writeHeaders

        ChannelFuture writeHeaders(ChannelHandlerContext ctx,
                           int streamId,
                           Http2Headers headers,
                           int padding,
                           boolean endStream,
                           ChannelPromise promise)
        将HEADERS帧写入远程端点。
        参数
        ctx - 用于书写的上下文。
        streamId - 要发送帧的流。
        headers - 要发送的标题。
        padding - 应添加的额外字节以遮盖真实内容大小。 必须介于0和256（含）之间。
        endStream - 指示这是否是流发送的最后一帧。
        promise - 写作的承诺。
        结果
        写作的未来。 Section 10.5.1声明如下： 
          The header block MUST be processed to ensure a consistent connection state, unless the connection is closed.
        如果这个调用修改了HPACK头部状态，你必须抛出连接错误。

        如果这一呼吁没有修改HPACK头状态，你可以自由地抛出一个流错误。
