    public class LengthFieldBasedFrameDecoder
extends ByteToMessageDecoder
    一个解码器，通过消息中的长度字段的值动态分割接收到的ByteBuf 。 当解码具有表示消息正文或整个消息长度的整数标题字段的二进制消息时，它特别有用。

    LengthFieldBasedFrameDecoder有许多配置参数，因此它可以解码长度字段的任何消息，这在专有的客户端 - 服务器协议中经常见到。 这里有一些例子会给你关于哪个选项做什么的基本想法。

    在偏移量为0的2字节长度字段中，不要剥离标题

    本例中长度字段的值为12 (0x0C) ，表示“HELLO，WORLD”的长度。 默认情况下，解码器假定长度字段表示长度字段后面的字节数。 因此，它可以用简单的参数组合来解码。 
      lengthFieldOffset   = 0
 lengthFieldLength   = 2
 lengthAdjustment    = 0
 initialBytesToStrip = 0 (= do not strip header)

 BEFORE DECODE (14 bytes)         AFTER DECODE (14 bytes)
 +--------+----------------+      +--------+----------------+
 | Length | Actual Content |----->| Length | Actual Content |
 | 0x000C | "HELLO, WORLD" |      | 0x000C | "HELLO, WORLD" |
 +--------+----------------+      +--------+----------------+

    带偏移量0的2字节长度字段，带头

    因为我们可以通过调用ByteBuf.readableBytes()来获取内容的长度，所以您可能希望通过指定initialBytesToStrip来剥离长度字段。 在这个例子中，我们指定了2 ，这与长度字段的长度相同，以去掉前两个字节。 
      lengthFieldOffset   = 0
 lengthFieldLength   = 2
 lengthAdjustment    = 0
 initialBytesToStrip = 2 (= the length of the Length field)

 BEFORE DECODE (14 bytes)         AFTER DECODE (12 bytes)
 +--------+----------------+      +----------------+
 | Length | Actual Content |----->| Actual Content |
 | 0x000C | "HELLO, WORLD" |      | "HELLO, WORLD" |
 +--------+----------------+      +----------------+

    在偏移量为0的2字节长度字段中，不剥离标题，长度字段表示整个消息的长度

    在大多数情况下，长度字段仅表示消息主体的长度，如前面的示例所示。 但是，在某些协议中，长度字段表示整个消息的长度，包括消息头。 在这种情况下，我们指定一个非零lengthAdjustment 。 由于此示例消息中的长度值始终大于2的正文长度， 因此我们将-2指定为lengthAdjustment进行补偿。 
      lengthFieldOffset   =  0
 lengthFieldLength   =  2
 lengthAdjustment    = -2 (= the length of the Length field)
 initialBytesToStrip =  0

 BEFORE DECODE (14 bytes)         AFTER DECODE (14 bytes)
 +--------+----------------+      +--------+----------------+
 | Length | Actual Content |----->| Length | Actual Content |
 | 0x000E | "HELLO, WORLD" |      | 0x000E | "HELLO, WORLD" |
 +--------+----------------+      +--------+----------------+

    3个字节的长度字段在5个字节的标题末尾，不要去掉标题

    以下消息是第一个示例的简单变体。 消息前面加了一个额外的头部值。 lengthAdjustment再次为零，因为在帧长度计算期间解码器始终考虑前置数据的长度。 
      lengthFieldOffset   = 2 (= the length of Header 1)
 lengthFieldLength   = 3
 lengthAdjustment    = 0
 initialBytesToStrip = 0

 BEFORE DECODE (17 bytes)                      AFTER DECODE (17 bytes)
 +----------+----------+----------------+      +----------+----------+----------------+
 | Header 1 |  Length  | Actual Content |----->| Header 1 |  Length  | Actual Content |
 |  0xCAFE  | 0x00000C | "HELLO, WORLD" |      |  0xCAFE  | 0x00000C | "HELLO, WORLD" |
 +----------+----------+----------------+      +----------+----------+----------------+

    3个字节的长度字段在5个字节的头部开头，不要去掉头部

    这是一个高级示例，它显示了长度字段和消息正文之间存在额外标题的情况。 您必须指定正数lengthAdjustment，以便解码器将额外的报头计入帧长度计算中。 
      lengthFieldOffset   = 0
 lengthFieldLength   = 3
 lengthAdjustment    = 2 (= the length of Header 1)
 initialBytesToStrip = 0

 BEFORE DECODE (17 bytes)                      AFTER DECODE (17 bytes)
 +----------+----------+----------------+      +----------+----------+----------------+
 |  Length  | Header 1 | Actual Content |----->|  Length  | Header 1 | Actual Content |
 | 0x00000C |  0xCAFE  | "HELLO, WORLD" |      | 0x00000C |  0xCAFE  | "HELLO, WORLD" |
 +----------+----------+----------------+      +----------+----------+----------------+

    2字节长度字段在4字节标题中间偏移1处，剥离第一个标题字段和长度字段

    这是上述所有示例的组合。 在长度字段之前有前置标题，在长度字段之后有额外的标题。 前置标题影响lengthFieldOffset ，额外标题影响lengthAdjustment 。 我们还指定了一个非零initialBytesToStrip去除帧中的长度字段和前置头。 如果您不想去除前置标题， 则可以为initialBytesToSkip指定0 。 
      lengthFieldOffset   = 1 (= the length of HDR1)
 lengthFieldLength   = 2
 lengthAdjustment    = 1 (= the length of HDR2)
 initialBytesToStrip = 3 (= the length of HDR1 + LEN)

 BEFORE DECODE (16 bytes)                       AFTER DECODE (13 bytes)
 +------+--------+------+----------------+      +------+----------------+
 | HDR1 | Length | HDR2 | Actual Content |----->| HDR2 | Actual Content |
 | 0xCA | 0x000C | 0xFE | "HELLO, WORLD" |      | 0xFE | "HELLO, WORLD" |
 +------+--------+------+----------------+      +------+----------------+

    2字节长度字段在偏移量为1的4字节头中间，剥离第一个头字段和长度字段，长度字段表示整个消息的长度

    让我们再来看看前面的例子。 与前面的示例唯一的区别是，长度字段代表整个消息的长度而不是消息正文，就像第三个示例一样。 我们必须计算HDR1的长度和长度为lengthAdjustment 。 请注意，我们不需要考虑HDR2的长度，因为长度字段已包含整个标题长度。 
      lengthFieldOffset   =  1
 lengthFieldLength   =  2
 lengthAdjustment    = -3 (= the length of HDR1 + LEN, negative)
 initialBytesToStrip =  3

 BEFORE DECODE (16 bytes)                       AFTER DECODE (13 bytes)
 +------+--------+------+----------------+      +------+----------------+
 | HDR1 | Length | HDR2 | Actual Content |----->| HDR2 | Actual Content |
 | 0xCA | 0x0010 | 0xFE | "HELLO, WORLD" |      | 0xFE | "HELLO, WORLD" |
 +------+--------+------+----------------+      +------+----------------+

        提取指定缓冲区的子区域。
