      • A

        public static final DnsRecordType A
        地址记录RFC 1035返回32位IPv4地址，最常用于将主机名映射到主机的IP地址，但也用于DNSBL，在RFC 1101中存储子网掩码等。

      • NS

        public static final DnsRecordType NS
        名称服务器记录RFC 1035委托DNS区域使用给定的权威名称服务器

      • CNAME

        public static final DnsRecordType CNAME
        规范名称记录RFC 1035一个名称的别名到另一个名称：DNS查找将继续通过用新名称重试查找。

      • SOA

        public static final DnsRecordType SOA
        RFC 1035和RFC 2308的[区域]的开始指定有关DNS区域的权威信息，包括主名称服务器，域管理员的电子邮件，域序列号以及与刷新区域有关的多个定时器。

      • PTR

        public static final DnsRecordType PTR
        指针记录RFC 1035指向规范名称的指针。 与CNAME不同，DNS处理不会继续，只是返回名称。 最常见的用途是实现反向DNS查找，但其他用途包括DNS-SD等。

      • MX

        public static final DnsRecordType MX
        邮件交换记录RFC 1035将域名映射到该域的邮件传输代理列表。

      • TXT

        public static final DnsRecordType TXT
        文本记录RFC 1035最初用于DNS记录中的任意人类可读文本。 然而，自20世纪90年代初以来，该记录更多地携带机器可读数据，如RFC 1464，机会加密，发送者策略框架，DKIM，DMARC DNS-SD等指定的数据。

      • RP

        public static final DnsRecordType RP
        负责人记录RFC 1183关于域名负责人的信息。 通常用@替换为@的电子邮件地址。

      • AFSDB

        public static final DnsRecordType AFSDB
        AFS数据库记录RFC 1183 AFS单元的数据库服务器的位置。 此记录通常由AFS客户端用于联系本地域外的AFS单元。 该记录的子类型由过时的DCE / DFS文件系统使用。

      • SIG

        public static final DnsRecordType SIG
        SIG（0）（RFC 2931）和TKEY（RFC 2930）中使用的签名记录RFC 2535签名记录。 RFC 3755指定RRSIG作为在DNSSEC中使用的SIG的替代品。

      • KEY

        public static final DnsRecordType KEY
        关键记录RFC 2535和RFC 2930仅用于SIG（0）（RFC 2931）和TKEY（RFC 2930）。 RFC 3445取消了应用程序密钥的使用，并将其用于DNSSEC。 RFC 3755指定DNSKEY作为DNSSEC内的替代品。 RFC 4025指定IPSECKEY作为IPsec使用的替代品。

      • AAAA

        public static final DnsRecordType AAAA
        IPv6地址记录RFC 3596返回一个128位的IPv6地址，最常用于将主机名映射到主机的IP地址。

      • LOC

        public static final DnsRecordType LOC
        位置记录RFC 1876指定与域名关联的地理位置。

      • SRV

        public static final DnsRecordType SRV
        服务定位器RFC 2782通用服务位置记录，用于较新的协议，而不是创建协议特定的记录（如MX）。

      • NAPTR

        public static final DnsRecordType NAPTR
        命名规范指针记录RFC 3403允许基于正则表达式的域名重写，然后可以将其用作URI，进一步的域名查找等。

      • KX

        public static final DnsRecordType KX
        主要eXchanger记录RFC 2230与某些加密系统（不包括DNSSEC）一起使用，以识别关联的域名的密钥管理代理。 请注意，这与DNS安全无关。 它是信息性的地位，而不是IETF的标准轨道。 它一直有限的部署，但仍在使用中。

      • CERT

        public static final DnsRecordType CERT
        证书记录RFC 4398存储PKIX，SPKI，PGP等

      • DNAME

        public static final DnsRecordType DNAME
        代理名称记录RFC 2672 DNAME为名称及其所有子名称创建别名，与CNAME不同，CNAME只标记其标签中的确切名称。 与CNAME记录一样，DNS查找将继续以新名称重试查找。

      • OPT

        public static final DnsRecordType OPT
        选项记录RFC 2671这是支持EDNS所需的伪DNS记录类型。

      • APL

        public static final DnsRecordType APL
        地址前缀列表记录RFC 3123指定不同地址系列的地址范围列表，例如CIDR格式。 实验。

      • DS

        public static final DnsRecordType DS
        代表签名人记录RFC 4034该记录用于标识委派区域的DNSSEC签名密钥。

      • SSHFP

        public static final DnsRecordType SSHFP
        SSH公钥指纹记录用于在DNS系统中发布SSH公钥主指纹的RFC 4255资源记录，以帮助验证主机的真实性。 RFC 6594定义了ECC SSH密钥和SHA-256哈希。 有关详细信息，请参阅IANA SSHFP RR参数注册表。

      • IPSECKEY

        public static final DnsRecordType IPSECKEY
        IPsec密钥记录RFC 4025可用于IPsec的密钥记录。

      • RRSIG

        public static final DnsRecordType RRSIG
        DNSSEC签名记录用于DNSSEC保护的记录集的RFC 4034签名。 使用与SIG记录相同的格式。

      • NSEC

        public static final DnsRecordType NSEC
        Next-Secure记录RFC 4034 DNSSEC的一部分，用于证明名称不存在。 使用与（过时的）NXT记录相同的格式。

      • DNSKEY

        public static final DnsRecordType DNSKEY
        DNS密钥记录RFC 4034 DNSSEC中使用的密钥记录。 使用与KEY记录相同的格式。

      • DHCID

        public static final DnsRecordType DHCID
        DHCP标识符记录RFC 4701与DHCP的FQDN选项一起使用。

      • NSEC3

        public static final DnsRecordType NSEC3
        NSEC记录版本3 RFC 5155 DNSSEC的扩展，允许在不允许区域漫游的情况下证明不存在名称。

      • NSEC3PARAM

        public static final DnsRecordType NSEC3PARAM
        NSEC3参数记录用于NSEC3的RFC 5155参数记录。

      • TLSA

        public static final DnsRecordType TLSA
        TLSA证书关联记录RFC 6698用于命名实体（DANE）的基于DNS的身份验证的记录。 RFC 6698定义TLSA DNS资源记录用于将TLS服务器证书或公钥与记录找到的域名相关联，从而形成'TLSA证书关联'。

      • HIP

        public static final DnsRecordType HIP
        主机标识协议记录RFC 5205分离IP地址的端点标识符和定位器角色的方法。

      • SPF

        public static final DnsRecordType SPF
        发件人策略框架记录RFC 4408作为将SPF数据存储在TXT记录中的替代方案，指定为SPF协议的一部分。 使用与早期TXT记录相同的格式。

      • TKEY

        public static final DnsRecordType TKEY
        密钥记录RFC 2930一种提供密钥资料的方法，与伴随的KEY RR中的公共密钥一起使用的TSIG使用。

      • TSIG

        public static final DnsRecordType TSIG
        事务签名记录RFC 2845可用于验证来自已批准客户端的动态更新，或者验证来自类似于DNSSEC的已批准递归名称服务器的响应。

      • IXFR

        public static final DnsRecordType IXFR
        增量区域转移记录RFC 1996请求区域传输给定区域，但仅与先前的序列号不同。 如果授权服务器由于配置或缺少必需的增量而无法满足请求，则可能会忽略此请求并发送完整（AXFR）响应。

      • AXFR

        public static final DnsRecordType AXFR
        权威性区域传输记录RFC 1035将整个区域文件从主名称服务器传输到辅助名称服务器。

      • ANY

        public static final DnsRecordType ANY
        所有缓存的记录RFC 1035返回名称服务器已知的所有类型的所有记录。 如果名称服务器没有关于名称的任何信息，则请求将被转发。 返回的记录可能不完整。 例如，如果名称同时包含A和MX，但名称服务器只缓存了A记录，则只会返回A记录。 有时称为ANY，例如在Windows nslookup和Wireshark中。

      • CAA

        public static final DnsRecordType CAA
        证书颁发机构授权记录RFC 6844 CA固定，约束主机/域的可接受CA.

      • TA

        public static final DnsRecordType TA
        DNSSEC信任机构记录N / A DNSSEC部署提议的一部分，不包含签名的DNS根目录。 有关详细信息，请参阅IANA数据库和Weiler规范。 使用与DS记录相同的格式。

      • DLV

        public static final DnsRecordType DLV
        DNSSEC旁视验证记录RFC 4431用于在DNS委派链之外发布DNSSEC信任锚。 使用与DS记录相同的格式。 RFC 5074描述了一种使用这些记录的方法。
