    • 构造方法详细信息

      • LzfEncoder

        public LzfEncoder()
        创建一个新的LZF编码器，其中包含用于底层数据访问的最佳可用方法。 如果可以在当前的JVM上使用它，它将会是“不安全”的实例。 在实现被动态加载时调用这个构造函数应该是安全的; 然而，对一些不规范的平台，可能需要使用LzfEncoder(boolean)与true PARAM。

      • LzfEncoder

        public LzfEncoder(boolean safeInstance)
        用指定的编码实例创建一个新的LZF编码器。
        参数
        safeInstance - 如果true编码器将使用仅使用标准JDK访问方法的ChunkEncoder ，并且应在所有Java平台和JVM上工作。 否则编码器将尝试使用高度优化的ChunkEncoder实现，该实现使用Sun JDK的Unsafe类（其他JDK也可能包含该类）。

      • LzfEncoder

        public LzfEncoder(int totalLength)
        创建具有指定编码块总长度的新LZF编码器。 如果知道您发送的消息的平均大小，您可以将其配置为更有效地编码数据流。
        参数
        totalLength - 预计要压缩的内容的总长度; 只对传出的消息小于最大块大小（64k），以优化编码散列表。

      • LzfEncoder

        public LzfEncoder(boolean safeInstance,
                  int totalLength)
        用指定的设置创建一个新的LZF编码器。
        参数
        safeInstance - 如果true编码器将使用仅使用标准JDK访问方法的ChunkEncoder ，并且应在所有Java平台和JVM上工作。 否则，编码器将尝试使用高度优化的ChunkEncoder实现，该实现使用Sun JDK的Unsafe类（其他JDK也可能包含该类）。
        totalLength - 预计要压缩的内容的总长度; 只对传出的消息小于最大块大小（64k），以优化编码散列表。
