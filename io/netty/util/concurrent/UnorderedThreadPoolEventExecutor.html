Class UnorderedThreadPoolEventExecutor

    EventExecutor ， EventExecutorGroup ，java.lang.Iterable < EventExecutor >，java.util.concurrent.Executor，java.util.concurrent.ExecutorService中，java.util.concurrent.ScheduledExecutorService 

    public final class UnorderedThreadPoolEventExecutor
extends java.util.concurrent.ScheduledThreadPoolExecutor
implements EventExecutor
    EventExecutor实现，因为可能有多个线程正在执行这些任务，所以对提交的任务执行的排序没有任何保证。 该实现对于不需要严格排序的协议非常有用。 因为它提供了使用它时不应该注意的顺序。

      • shutdownGracefully

        public Future<?> shutdownGracefully(long quietPeriod,
                                    long timeout,
                                    java.util.concurrent.TimeUnit unit)
        从界面： EventExecutorGroup复制的描述
        该执行者表示该调用者希望执行者被关闭。 一旦调用此方法， EventExecutorGroup.isShuttingDown()开始返回true ，并且执行程序准备关闭它。 与EventExecutorGroup.shutdown()不同，正常关机可确保在关闭之前没有任何任务被提交“安静期” （通常是几秒钟）。 如果任务在安静期提交，则保证被接受，安静期将重新开始。
        Specified by:
        shutdownGracefully ，界面 EventExecutorGroup
        参数
        quietPeriod - 文档中描述的安静期
        timeout - 无论任务是否在静默期间提交，等待执行程序为 EventExecutorGroup.shutdown()的最长时间
        unit - quietPeriod和 timeout的单位
        结果
        EventExecutorGroup.terminationFuture()
